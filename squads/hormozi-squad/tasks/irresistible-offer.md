# Task: Criar Oferta Irresistivel (Grand Slam Offer)

## Objetivo
Construir uma oferta tão boa que o cliente se sente idiota em recusar — com value stack completo, garantia ousada e preço que parece ridiculamente acessível comparado ao valor.

## Inputs Necessários
```
produto_central: [o que você entrega]
avatar: [quem compra]
dream_outcome: [resultado final que o cliente realmente quer]
preco_atual_ou_desejado: [R$X]
garantia_maxima_possivel: [o que você pode prometer com segurança]
recursos_disponiveis: [o que você pode incluir como bônus]
concorrentes_referencia: [o que o mercado cobra por soluções similares]
```

## Processo de Construção (Hormozi Method)

### PASSO 1: Dream Outcome
Defina o resultado com máxima especificidade:
- Resultado mensurável: "X em Y dias/semanas/meses"
- Resultado emocional: "sentir-se como..."
- Resultado de status: "ser reconhecido como..."
- Resultado de liberdade: "nunca mais ter que..."

### PASSO 2: Mapeamento de Obstáculos
Liste TODOS os obstáculos entre o cliente e o Dream Outcome:
```
Obstáculo 1: [falta de conhecimento sobre X]
Obstáculo 2: [falta de tempo para Y]
Obstáculo 3: [medo de Z]
Obstáculo 4: [custo de W]
Obstáculo 5: [não saber por onde começar]
...
```

### PASSO 3: Soluções para Cada Obstáculo
Para cada obstáculo, criar um componente da oferta:
```
Obstáculo 1 → [Nome do componente] — entrega [X resultado específico]
Obstáculo 2 → [Nome do componente] — resolve [Y em Z tempo]
...
```

### PASSO 4: Value Stack
```
COMPONENTE PRINCIPAL: [Nome aspiracional]
Valor de mercado: R$[X] — Justificativa: [por que vale isso]

BÔNUS 1: [Nome aspiracional]
Valor de mercado: R$[X] — Justificativa: [por que vale isso]

BÔNUS 2: [Nome aspiracional]
Valor de mercado: R$[X] — Justificativa: [por que vale isso]

BÔNUS 3: [Nome aspiracional]
Valor de mercado: R$[X] — Justificativa: [por que vale isso]

BÔNUS SURPRESA: [Adicionar algo que não estava anunciado]
Valor de mercado: R$[X]

━━━━━━━━━━━━━━━━━━━━━━━
VALOR TOTAL: R$[soma]
VOCÊ INVESTE HOJE: R$[preco]
ECONOMIA: R$[diferença] ([percentual]%)
```

### PASSO 5: Garantia
Tipos de garantia (do menos para o mais ousado):
- **Satisfação garantida**: "Gostou ou devolvemos seu dinheiro em [prazo]"
- **Resultado garantido**: "Se você não [resultado específico] em [prazo], devolvemos tudo"
- **Garantia anti-risco total**: "Se não funcionar, devolvemos o dinheiro E você fica com todo o material"
- **Garantia de resultado + compensação**: "Se não funcionar, devolvemos o dobro"

Escolher a mais ousada que o negócio pode honrar de forma sustentável.

### PASSO 6: Nome da Oferta
A oferta precisa de um nome que:
- Comunica o resultado (não o método)
- Tem um número (ex: "Sistema 30 Dias")
- Tem um adjetivo de resultado (ex: "Garantido", "Comprovado", "Definitivo")
- É memorável e proprietário

Formato: `[Resultado] em [Prazo] [Sistema/Método/Programa]`

## Output Esperado
- Grand Slam Offer completa com todos os 6 passos
- Value stack formatado para uso em página de vendas
- 3 opções de nome para a oferta
- Script de apresentação oral da oferta (2 minutos)
- Versão "teaser" da oferta para anúncio (sem revelar o preço)
